Overview

Project Overview

Built as an interview task, this Laravel application lets users upload a document (PDF, DOCX, or TXT), and then uses AI via OpenRouter to intelligently summarize and reformat the content — producing an export-ready result as a PDF or Word file.

The real challenge was designing smart prompts that bridge raw text analysis with professional visual formatting — headings, bullet points, symbols, and export-ready structure. The system supports multiple AI models (GPT-4, Claude, Mistral, and more) via OpenRouter, making it model-agnostic.

How It Works

AI Processing Pipeline
01
Upload File
User uploads a PDF, DOCX, or TXT file after authenticating via Google OAuth
02
Parse & Extract Text
PdfParser + PdfToText extract raw content from PDFs; PhpWord handles DOCX files
03
Smart Prompt Engineering
A carefully crafted prompt instructs the AI to understand the language, summarize, and reformat with structure and visual hierarchy
04
AI Processing via OpenRouter
Sends the prompt to OpenRouter — supports GPT-4, Claude, Mistral, and more. Model-agnostic by design
05
Export Result
AI output is rendered and exported as a downloadable PDF (mPDF / DomPDF) or Word document (PhpWord)

Features

Key Features
  • Google OAuth — Passwordless login via Laravel Socialite. No registration forms, no passwords
  • Multi-format Upload — Accepts PDF, DOCX, and TXT files with automatic format detection
  • AI Processing — Summarizes content and reformats it with headings, key points, symbols, and professional structure
  • Language Detection — The AI understands the file language automatically — Arabic or English
  • PDF Export — Download result as a styled PDF via mPDF or Barryvdh DomPDF
  • Word Export — Download result as a formatted DOCX file via PhpWord
  • Responsive UI — Clean, mobile-friendly interface built with Bootstrap 5

Technical Challenges

What Made It Challenging
Prompt Engineering — Designing prompts that reliably produce structured, visually formatted output (not just plain text) was the hardest part. Each prompt iteration changed the quality of the final document significantly.
Multi-format Parsing — Each file format (PDF, DOCX, TXT) requires a different extraction strategy. PDFs especially vary wildly in encoding — scanned vs text-based required separate handling.
Model-agnostic Design — Abstracting OpenRouter so any model (GPT-4, Claude, Mistral) can be swapped without changing application logic required a clean service layer interface.
Export Fidelity — Converting AI markdown-style output into properly formatted Word/PDF required careful mapping between AI response tokens and PhpWord/mPDF rendering APIs.
